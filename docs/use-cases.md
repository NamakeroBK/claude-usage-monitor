## 問題点・課題

### 🔴 重大な問題

#### P1: セッションキーの有効期限
**問題:** セッションキーの有効期限が不明。突然失効する可能性
**影響:** 認証エラーでアプリが動かなくなる
**対策案:**
- 認証エラー時の自動検知
- 再設定を促す通知
- セッションキーの自動更新（可能なら）

#### P2: API変更リスク
**問題:** Claude.aiの内部APIは非公式。いつ変更されるかわからない
**影響:** アプリが突然動かなくなる
**対策案:**
- エラーハンドリングの強化
- APIレスポンスの検証
- 変更検知時の通知

#### P3: レート制限
**問題:** 短時間に大量リクエストするとブロックされる可能性
**影響:** アカウント停止リスク
**対策案:**
- ポーリング間隔を控えめに（30秒〜1分）
- エラー時のバックオフ
- 手動リフレッシュの制限

### 🟡 中程度の問題

#### P4: Cookie取得の手間
**問題:** DevToolsからセッションキーをコピーするのが面倒
**影響:** 導入障壁が高い
**対策案:**
- 詳しい手順ガイド
- スクリーンショット付き説明
- （将来）ブラウザ拡張との連携

#### P5: 週間使用量が取得できない？
**問題:** 現在のAPIでは5時間ウィンドウのみ。週間データは別？
**影響:** Pro/Max/Teamの週間制限が見えない
**対策案:**
- 追加のAPI調査が必要
- 参照プロジェクトのコード確認

#### P6: Windows起動時の自動起動
**問題:** レジストリ操作が必要
**影響:** ユーザーの許可が必要
**対策案:**
- オプトイン設定
- レジストリへの適切なパス登録

### 🟢 軽微な問題

#### P7: ダークモードのみ
**問題:** UIがダークテーマ固定
**影響:** 明るいテーマ好みのユーザーには不便
**対策案:**
- Windowsテーマに追従するオプション

#### P8: 多言語対応なし
**問題:** 英語UIのみ
**影響:** 日本語ユーザーには少し不便
**対策案:**
- リソースファイルでの多言語化

## 追加調査結果（2026-02-02）

### 調査: 週間使用量のAPI

**結論:** 現在のところ、公式な週間使用量APIは見つからない。

**参照プロジェクトの実装:**
- **Chrome拡張 (lugia19):** トークンを自前でカウント。会話データを取得して計算。
- **macOS版 (hamed-elfayome):** `/api/organizations/{orgId}/usage` から5時間データのみ取得。週間データのフィールドは存在するが、取得元不明（未実装の可能性）。

**APIレスポンス構造（確認済み）:**
```json
{
  "five_hour": {
    "utilization": 45,
    "resets_at": "2026-02-02T21:00:00.000Z"
  }
}
```

**MVP方針:** 5時間ウィンドウを確実に表示。週間データは後で対応。
